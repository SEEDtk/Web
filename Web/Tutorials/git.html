Getting Started Using GIT for Code Development in SEEDtk

Projects: Where to Go to Find Code

Within the SEEDtk Project we have defined a set of projects. Each project directory can be found in SEEDtk/GIT/project-name. Each project directory will include two subdirectories

  1. lib will contain perl modules and
  2. scripts will contain perl scripts.

The following projects form the core of SEEDtk:

  • tbltools includes the code for svc-scripts, as well the general databse access scripts (e.g., all_entities, get_entities, and get_relationship). When you code an svc-script, there are usually two components:
    1. A core subroutine will be placed in tbltools/lib/STKServices.pm which can be invoked from any Perl script to gain the basic functionality via a subroutine call.
    2. Then the actual svc-script is put into tbltools/scripts.
  • RASTtk is the project used to hold code that is shared with the old SEED world.
  • kernel contains the collection of tools that make up the SEEDtk.

Most of the code a user or developer adds will go into the kernel project with a few occasional svc-scripts in tbltools.

Maintaining the Source in GIT

The original SEED Project used CVS, and it worked remarkably well. Users developing code in SEEDtk will be using GIT. They can elect to use Eclipse (get help from Bruce Parrello) or they can use a much more primitive style analogous to the old CVS commands. Here we summarize the command line style.

  1. Code development can be done anywhere, as long as you source usr-env.sh using something like 
          source /home/SEEDtk/GIT/SEEDtk/user-env.sh
    which assumes that you set up SEEDtk in /home/SEEDtk.
  2. Before beginning to debug a new program, we suggest that you run 
          pull-all
    which pulls all changes others have made.
  3. Code and debug your program.
  4. Copy your new script to the appropriate scripts directory, usually something like 
      mv CODE.pl /home/SEEDtk/GIT/kernel/scripts
  cd /home/SEEDtk/GIT/kernel/scripts
    or 
      mv CODE.pl /home/SEEDtk/GIT/tbltools/scripts
  cd /home/SEEDtk/GIT/tbltools/scripts
  5. then run 
      git -add CODE.pl
  git commit -m "added code CODE.pl"
  git pull
  git push
    Make sure that you check for errors on the pull before running the push. If you get one, you probably have a conflict. To fix this, you go to CODE.pl, fix it, rerun the add, commit, and push.

These are just minimal notes to get you started, but if you are careful, you should be able to fairly painlessly develop code. Eventually, you will probably get into errors that are difficult to resolve. When you get there, ask for help.
