This document describes the procedure for installing a tutorial web page on this site. Eventually, we wish all of the important SEEDtk tasks to be documented in a tutorial here.

To start, you will need to create an empty HTML file with the following skeleton.

        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8" />
        <title>... tutorial title ... </title>
        <link href="../css/Basic.css" rel="stylesheet" type="text/css">
        </head>
        <body>
        <div id="Dump">
            ... your text here ...
        </div>
        </body>
        </html>

The file should be stored in the Web/Web/Tutorials directory of your SEEDtk code folder, where it will be in the scope of the GIT source control system. If you are unsure of the location of this directory, use the tut-dir command.

Note that the text inside the <title> tag above will be shown in the master tutorial index, so it is important to choose a descriptive one.

Once the tutorial HTML file is complete, it will automatically appear on your localhost web site in the Tutorials list.

The tutorial must now be updated in the master repository. Switch to the tutorials directory and use the following commands.

git add ...filename...
git pull
git commit -m 'tutorial update'
git push

The first command (add) tells GIT that you have modified the named file (which in this case is your HTML tutorial). The second command (pull) insures your web files are up-to-date. It is not always necessary, but it reduces the incidence of the most common errors in the final command. The third command (commit) stores the updated tutorial in your local source control system. The final command (push) copies the changes to the master source control system.
